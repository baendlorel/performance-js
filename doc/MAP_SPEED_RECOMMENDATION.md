# 🎯 Map 遍历方式选择指南

## 快速答案

```javascript
const map = new Map([...Array(1000).keys()]);

// ✅ 最佳选择（大部分情况）
map.forEach((value, key) => {
  // 你的操作
});

// ✅ 高性能选择（大量重复遍历）
const entries = [...map]; // 只展开一次
for (let i = 0; i < 100000; i++) {
  for (let j = 0; j < entries.length; j++) {
    const [key, value] = entries[j];
    // 你的操作
  }
}

// ❌ 避免使用
for (const [key, value] of map) {
  // 慢 4-5 倍，无任何优势
}
```

## 性能对比表

| 操作 | 相对性能 | 适用场景 |
|------|----------|----------|
| `map.forEach()` | ⚡⚡⚡⚡⚡ | 通用、简洁、大部分场景 |
| `for + [...map]` | ⚡⚡⚡⚡ | 大量重复遍历同一 Map |
| `for...of map` | ⚡ | 不推荐 |
| `for...of entries()` | ⚡ | 不推荐 |

## 何时使用哪种方式？

### 使用 `map.forEach()` 当：
✅ 遍历次数 < 10,000
✅ 追求代码简洁性
✅ Map 内容会变化
✅ 只遍历一次

### 使用 `for + [...map]` 当：
✅ 遍历次数 > 10,000
✅ Map 内容固定
✅ 性能关键路径
✅ 需要索引访问

### 避免 `for...of map` 当：
❌ 它比 forEach 慢 4-5 倍
❌ 没有任何使用优势
❌ 代码可读性也不更好

## 原理解释

### 为什么 `forEach` 适合少量遍历？
- 直接在 Map 内部迭代，零额外开销
- 不需要创建中间数组

### 为什么 `for + [...map]` 适合大量遍历？
- `[...map]` 的展开成本被多次遍历摊销
- 数组索引访问比 Map 迭代器更快
- 现代 JS 引擎对数组访问优化更好

### 为什么 `for...of` 慢？
- 迭代器协议开销
- 每次迭代都要调用 next()
- 没有 forEach 的内部优化

## 记忆口诀

> 少量用 forEach，大量用展开，永远别用 for...of
